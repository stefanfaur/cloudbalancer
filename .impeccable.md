## Design Context

### Users
Mix of technical users — DevOps/platform engineers managing infrastructure and monitoring task execution, data scientists submitting Python/Docker jobs, and university reviewers evaluating the project as a portfolio piece. The dashboard must be genuinely functional for operational use while impressive enough to demonstrate engineering quality to professors and peers.

### Brand Personality
**Professional, Precise, Reliable** — enterprise infrastructure feel. CloudBalancer should convey the confidence and trustworthiness of production-grade systems tooling. Think operational dashboards where clarity and information density matter more than playfulness.

### Aesthetic Direction
- **Theme:** Dark mode primary — dark backgrounds with vibrant accents, standard for DevOps/monitoring tools
- **References:** Grafana, Datadog — dense information displays, real-time charts, dark panels, operational dashboards
- **Anti-references:** Overly minimal consumer apps, playful/casual aesthetics, marketing-heavy layouts with too much whitespace
- **Typography:** Monospace for data/metrics/logs, clean sans-serif for UI chrome. Prioritize readability at small sizes for dense data displays
- **Color:** Vibrant semantic accents on dark surfaces — greens for healthy/success, reds/oranges for errors/warnings, blues for informational states. Avoid oversaturation; muted accents with bright highlights on hover/focus
- **Density:** Information-dense by default. Users are technical and prefer seeing more data over decorative spacing

### Design Principles
1. **Clarity over decoration** — Every pixel serves a purpose. Prefer data density and clear hierarchy over ornamental elements.
2. **Real-time confidence** — Live-updating data (logs, metrics, task states) should feel immediate and trustworthy. Stale data must be visually distinct.
3. **Operational first** — Design for the user who's debugging a failed task at 2am. Fast scanning, clear status indicators, minimal clicks to critical information.
4. **Progressive disclosure** — Show summary by default, drill into detail on demand. Don't overwhelm, but don't hide.
5. **Consistent semantics** — Colors, icons, and patterns mean the same thing everywhere. Green is always healthy, red is always error, amber is always warning.
