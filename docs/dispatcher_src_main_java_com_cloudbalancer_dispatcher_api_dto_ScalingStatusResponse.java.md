# File: dispatcher/src/main/java/com/cloudbalancer/dispatcher/api/dto/ScalingStatusResponse.java

## Overview

The `ScalingStatusResponse` is a Java `record` used as a Data Transfer Object (DTO) to encapsulate the current state of the cloud scaling infrastructure. It provides a snapshot of the worker pool distribution, the active scaling policy, the most recent scaling decision made by the system, and the remaining cooldown period.

## Public API

### `ScalingStatusResponse` (Record)

| Field | Type | Description |
| :--- | :--- | :--- |
| `workerCount` | `int` | The total number of workers currently registered in the system. |
| `activeWorkerCount` | `int` | The number of workers currently in an active, ready-to-process state. |
| `drainingWorkerCount` | `int` | The number of workers currently in a draining state, finishing tasks before termination. |
| `policy` | `ScalingPolicy` | The configuration object defining the current scaling rules. |
| `lastDecision` | `ScalingDecision` | The most recent scaling action or recommendation generated by the dispatcher. |
| `cooldownRemainingSeconds` | `long` | The duration in seconds remaining before the next scaling action can be triggered. |

## Dependencies

This class relies on the following domain models from the `com.cloudbalancer.common.model` package:

*   **`ScalingPolicy`**: Defines the thresholds and rules governing the scaling behavior.
*   **`ScalingDecision`**: Represents the outcome of the scaling logic (e.g., scale up, scale down, or hold).

## Usage Notes

*   **Immutability**: As a Java `record`, this object is immutable. Once instantiated, its state cannot be modified, making it thread-safe for concurrent API responses.
*   **API Contract**: This DTO is intended for use in REST API responses to provide monitoring tools and dashboards with real-time visibility into the dispatcher's scaling logic.
*   **Cooldown Logic**: The `cooldownRemainingSeconds` field is critical for clients to understand why the system might not be reacting to load changes immediately, as it indicates the system is currently in a stabilization period.
