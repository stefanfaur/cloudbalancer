# File: common/src/main/java/com/cloudbalancer/common/agent/AgentEvent.java

## Overview

The `AgentEvent` interface is a sealed interface that defines the contract for all events generated by the CloudBalancer agent regarding worker lifecycle management. It provides a polymorphic structure for serializing and deserializing various worker-related state changes, such as startup and shutdown events, using Jackson annotations.

## Public API

### Interface: `AgentEvent`

The base interface for all agent-related events.

#### Methods

*   **`String agentId()`**: Returns the unique identifier of the agent that generated the event.
*   **`String workerId()`**: Returns the unique identifier of the worker associated with the event.

### Implementations (Records)

*   **`WorkerStartedEvent`**: Indicates a worker has successfully started.
    *   Fields: `agentId`, `workerId`, `containerId`, `timestamp`.
*   **`WorkerStartFailedEvent`**: Indicates a failure during the worker startup process.
    *   Fields: `agentId`, `workerId`, `reason`, `timestamp`.
*   **`WorkerStoppedEvent`**: Indicates a worker has been successfully stopped.
    *   Fields: `agentId`, `workerId`, `timestamp`.
*   **`WorkerStopFailedEvent`**: Indicates a failure during the worker shutdown process.
    *   Fields: `agentId`, `workerId`, `reason`, `timestamp`.

## Dependencies

*   `com.fasterxml.jackson.annotation.JsonSubTypes`: Used for polymorphic type handling during JSON serialization.
*   `com.fasterxml.jackson.annotation.JsonTypeInfo`: Configures the JSON metadata (property `eventType`) used to distinguish between event types.
*   `java.time.Instant`: Used for capturing precise event timestamps.

## Usage Notes

*   **Polymorphism**: This interface uses Jackson's `@JsonTypeInfo` and `@JsonSubTypes` annotations. When serializing or deserializing, ensure the JSON payload includes the `eventType` field, which maps to the defined names (e.g., `WORKER_STARTED`, `WORKER_STOPPED`).
*   **Sealed Interface**: The interface is `sealed`, meaning only the explicitly defined records within the file are permitted to implement it. This ensures type safety and exhaustive pattern matching when processing events.
*   **Immutability**: All implementations are defined as Java `record` types, ensuring that event data is immutable once created.
