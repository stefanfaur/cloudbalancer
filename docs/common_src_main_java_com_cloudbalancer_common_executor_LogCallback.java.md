# File: common/src/main/java/com/cloudbalancer/common/executor/LogCallback.java

## Overview

The `LogCallback` interface is a functional interface within the `com.cloudbalancer.common.executor` package. It defines a standardized contract for processing log streams generated by task executions. By providing a callback mechanism, it allows the system to decouple the log generation process from the log handling logic, enabling features such as real-time streaming, rate-limiting, or persistent storage of task outputs.

## Public API

### `LogCallback` (interface)

A functional interface designed to be implemented by any component responsible for consuming log output from a task.

#### `onLogLine(String line, boolean isStderr, Instant timestamp)`

Invoked when a new line of log output is produced by a task.

*   **Parameters:**
    *   `line` (`String`): The content of the log message.
    *   `isStderr` (`boolean`): A flag indicating whether the log originated from the standard error stream (`true`) or standard output (`false`).
    *   `timestamp` (`Instant`): The precise time at which the log line was captured.

## Dependencies

*   `java.time.Instant`: Used to provide high-precision temporal context for log entries.

## Usage Notes

*   **Functional Interface**: As a `@FunctionalInterface`, `LogCallback` can be implemented using lambda expressions, making it highly convenient for simple logging requirements.
*   **Implementation Example**: For complex requirements, such as handling high-volume log streams, consider implementing this interface within a service (e.g., `RateLimitedLogCallback`) to manage buffering or throttling.
*   **Integration**: This interface is primarily used by the task execution infrastructure to route logs from worker processes to the dispatcher or persistent storage layers. Ensure that implementations are thread-safe if the underlying executor processes logs concurrently.
