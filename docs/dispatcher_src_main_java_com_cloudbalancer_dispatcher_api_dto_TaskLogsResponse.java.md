# File: dispatcher/src/main/java/com/cloudbalancer/dispatcher/api/dto/TaskLogsResponse.java

## Overview

The `TaskLogsResponse` class is a lightweight Data Transfer Object (DTO) used to encapsulate the output logs generated by a task execution within the cloud balancer system. It is implemented as a Java `record`, providing an immutable structure for transporting standard output and standard error streams from the dispatcher to the requesting client.

## Public API

### `TaskLogsResponse`

```java
public record TaskLogsResponse(String stdout, String stderr) {}
```

#### Constructors
*   **`TaskLogsResponse(String stdout, String stderr)`**: Constructs a new response object containing the captured standard output and standard error strings.

#### Accessor Methods
*   **`stdout()`**: Returns the standard output (`stdout`) captured from the task execution.
*   **`stderr()`**: Returns the standard error (`stderr`) captured from the task execution.

## Dependencies

This class is a standard Java `record` and has no external dependencies beyond the Java Development Kit (JDK) 16 or higher.

## Usage Notes

*   **Immutability**: As a `record`, instances of this class are immutable. Once created, the log contents cannot be modified.
*   **Serialization**: This DTO is intended to be serialized (typically to JSON) when returned by the dispatcher's API endpoints. Ensure that the consuming service is configured to handle the `stdout` and `stderr` fields appropriately.
*   **Nullability**: Depending on the task execution status, these fields may contain empty strings or `null` values if no output was produced. Consumers should implement null-safe checks when processing these fields.
